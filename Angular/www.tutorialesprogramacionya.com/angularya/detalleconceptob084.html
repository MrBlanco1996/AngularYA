61 - Estructura y nombres de archivos y carpetas de un proyecto Angular - Carpetas '<Nombre del proyecto>\dist'

Las carpetas '<Nombre del proyecto>\dist' no se generan inmediatamente al crear un proyecto Angular, sino cuando compilamos por primera vez:

ng build --prod

Luego de compilar el proyecto Angular por primera vez podemos ver que se han generado las carpetas 'proyecto37\dist' dependiendo de la raiz:


Dentro de la carpeta 'proyecto37' se localizan todos los archivos y carpeta 'assets' que deben ser subidos al servidor web.

  • index.html El archivo index.html no es exactamente el mismo archivo que se encuentra en la raiz del proyecto, sino con los script necesarios de la aplicación Angular:

    <!doctype html>
<html lang="en">

<head>
  <meta charset="utf-8">
  <title>Proyecto037</title>
  <base href="/">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <link rel="icon" type="image/x-icon" href="favicon.ico">
<link rel="stylesheet" href="styles.3ff695c00d717f2d2a11.css"></head>

<body>
  <app-root></app-root>
<script src="runtime-es2015.0811dcefd377500b5b1a.js" type="module"></script>
<script src="runtime-es5.0811dcefd377500b5b1a.js" nomodule defer></script>
<script src="polyfills-es5.277e2e1d6fb2daf91a5c.js" nomodule defer></script>
<script src="polyfills-es2015.ca64e4516afbb1b890d5.js" type="module"></script>
<script src="main-es2015.b68fe9b6ed7f0fc1899b.js" type="module"></script>
<script src="main-es5.b68fe9b6ed7f0fc1899b.js" nomodule defer></script>
</body>

</html>

    Si analizamos el código añadido podemos ver que se han agregado etiquetas 'script'.Cada etiqueta de script tiene un atributo type="module" o nomodule defer.

    Los navegadores con soporte nativo para módulos ES (navegadores modernos) solo cargan los scripts con el atributo module e ignoran los scripts con el atributo nomodule.
    En cambio los navegadores heredados solo cargan los scripts con el atributos nomodule e ignoran las etiquetas de script con el tipo module.

    Angular genera una aplicación más eficiente y pequeña para los navegadores modernos, podemos ver que los archivos que se ejecutarán tienen un tamaño menor:


    Si sumamos vemos que en los navegadores modernos el sitio web tiene un peso de: 141Kb.

    Significa que si estamos accediendo al sitio web con un navegador antiguo los script que se descargarán son:


    Si sumamos vemos que en los navegadores antiguos el sitio web tiene un peso de: 253Kb.

    Es decir de los 6 archivos *.js solo se recuperan 3 de ellos, y eso depende del navegador donde se este ejecutando la aplicación.

  • styles.xxxxxxxx.css Contiene una copia de la hoja de estilo global del proyecto (borra saltos de línea y espacios en blanco para reducir su tamaño), recordemos que se almacena en la carpeta 'src' el archivo 'styles.css'.

  • favicon.ico Es la copia del archivo que se encuentra en la carpeta raiz del proyecto.
